## 🎯 Principios SOLID

Los principios SOLID son 5 reglas fundamentales para escribir código orientado a objetos de calidad.

## 📐 S - Single Responsibility (Responsabilidad Única)

Cada clase debe tener **una sola razón para cambiar**.

### ❌ Mal:
```python
class APIClient:
    def __init__(self, api_key):
        self.api_key = api_key
    
    def hacer_peticion(self, url):
        # Conectar a API
        pass
    
    def guardar_en_bd(self, datos):
        # Conectar a BD
        # Insertar datos
        pass
    
    def enviar_email_notificacion(self, email):
        # Conectar a servidor email
        # Enviar email
        pass
    
    def generar_reporte_pdf(self, datos):
        # Crear PDF
        pass
```
**Problema:** APIClient hace demasiadas cosas (API, BD, Email, PDF)

### ✅ Bien:
```python
class APIClient:
    def __init__(self, api_key):
        self.api_key = api_key
    
    def hacer_peticion(self, url):
        # Solo maneja peticiones API
        pass

class RepositorioDatos:
    def guardar(self, datos):
        # Solo maneja persistencia
        pass

class ServicioEmail:
    def enviar_notificacion(self, email, mensaje):
        # Solo maneja emails
        pass

class GeneradorReportes:
    def generar_pdf(self, datos):
        # Solo genera reportes
        pass
```

## 🔓 O - Open/Closed (Abierto/Cerrado)

**Abierto para extensión, cerrado para modificación**

### ❌ Mal:
```python
class ProcesadorPagos:
    def procesar(self, metodo, monto):
        if metodo == "tarjeta":
            # Lógica tarjeta
            return self._procesar_tarjeta(monto)
        elif metodo == "paypal":
            # Lógica PayPal
            return self._procesar_paypal(monto)
        elif metodo == "stripe":
            # Lógica Stripe
            return self._procesar_stripe(monto)
        # ❌ Para añadir nuevo método, hay que modificar esta clase
```

### ✅ Bien:
```python
from abc import ABC, abstractmethod

class MetodoPago(ABC):
    @abstractmethod
    def procesar(self, monto):
        pass

class PagoTarjeta(MetodoPago):
    def procesar(self, monto):
        print(f"Procesando {monto}€ con tarjeta")
        return True

class PagoPayPal(MetodoPago):
    def procesar(self, monto):
        print(f"Procesando {monto}€ con PayPal")
        return True

class PagoStripe(MetodoPago):
    def procesar(self, monto):
        print(f"Procesando {monto}€ con Stripe")
        return True

class ProcesadorPagos:
    def procesar(self, metodo_pago: MetodoPago, monto):
        return metodo_pago.procesar(monto)

# ✅ Para añadir nuevo método, solo creas nueva clase
class PagoBitcoin(MetodoPago):
    def procesar(self, monto):
        print(f"Procesando {monto}€ con Bitcoin")
        return True
```

## 🔄 L - Liskov Substitution (Sustitución de Liskov)

Las subclases deben poder sustituir a sus clases base sin romper funcionalidad.

### ❌ Mal:
```python
class Rectangulo:
    def __init__(self, ancho, alto):
        self.ancho = ancho
        self.alto = alto
    
    def area(self):
        return self.ancho * self.alto

class Cuadrado(Rectangulo):
    def __init__(self, lado):
        super().__init__(lado, lado)
    
    def set_ancho(self, ancho):
        self.ancho = ancho
        self.alto = ancho  # ❌ Rompe el contrato de Rectangulo
```

### ✅ Bien:
```python
from abc import ABC, abstractmethod

class Forma(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangulo(Forma):
    def __init__(self, ancho, alto):
        self.ancho = ancho
        self.alto = alto
    
    def area(self):
        return self.ancho * self.alto

class Cuadrado(Forma):
    def __init__(self, lado):
        self.lado = lado
    
    def area(self):
        return self.lado ** 2
```

## 🔌 I - Interface Segregation (Segregación de Interfaces)

No obligar a implementar métodos que no se usan.

### ❌ Mal:
```python
class Dispositivo(ABC):
    @abstractmethod
    def encender(self):
        pass
    
    @abstractmethod
    def apagar(self):
        pass
    
    @abstractmethod
    def recargar_bateria(self):
        pass

class Lampara(Dispositivo):
    def encender(self):
        print("Lámpara encendida")
    
    def apagar(self):
        print("Lámpara apagada")
    
    def recargar_bateria(self):
        # ❌ Una lámpara de pared no tiene batería
        pass
```

### ✅ Bien:
```python
class Encendible(ABC):
    @abstractmethod
    def encender(self):
        pass
    
    @abstractmethod
    def apagar(self):
        pass

class Recargable(ABC):
    @abstractmethod
    def recargar_bateria(self):
        pass

class Lampara(Encendible):
    def encender(self):
        print("Lámpara encendida")
    
    def apagar(self):
        print("Lámpara apagada")

class Smartphone(Encendible, Recargable):
    def encender(self):
        print("Smartphone encendido")
    
    def apagar(self):
        print("Smartphone apagado")
    
    def recargar_bateria(self):
        print("Recargando batería")
```

## 🔁 D - Dependency Inversion (Inversión de Dependencias)

Depender de abstracciones, no de implementaciones concretas.

### ❌ Mal:
```python
class BaseDatosMySQL:
    def conectar(self):
        print("Conectando a MySQL")
    
    def guardar(self, datos):
        print(f"Guardando {datos} en MySQL")

class Aplicacion:
    def __init__(self):
        self.bd = BaseDatosMySQL()  # ❌ Dependencia directa
    
    def procesar(self, datos):
        self.bd.guardar(datos)
```

### ✅ Bien:
```python
from abc import ABC, abstractmethod

class BaseDatos(ABC):
    @abstractmethod
    def conectar(self):
        pass
    
    @abstractmethod
    def guardar(self, datos):
        pass

class BaseDatosMySQL(BaseDatos):
    def conectar(self):
        print("Conectando a MySQL")
    
    def guardar(self, datos):
        print(f"Guardando {datos} en MySQL")

class BaseDatosPostgreSQL(BaseDatos):
    def conectar(self):
        print("Conectando a PostgreSQL")
    
    def guardar(self, datos):
        print(f"Guardando {datos} en PostgreSQL")

class Aplicacion:
    def __init__(self, bd: BaseDatos):  # ✅ Dependencia de abstracción
        self.bd = bd
    
    def procesar(self, datos):
        self.bd.guardar(datos)

# Uso
app1 = Aplicacion(BaseDatosMySQL())
app2 = Aplicacion(BaseDatosPostgreSQL())
```

## 🚫 Code Smells Comunes

### **1. Clases Demasiado Grandes**
```python
# ❌ 500+ líneas en una clase
# ✅ Dividir en clases más pequeñas con responsabilidades claras
```

### **2. Métodos Demasiado Largos**
```python
# ❌ Método con 50+ líneas
# ✅ Extraer a métodos privados auxiliares
```

### **3. Demasiados Parámetros**
```python
# ❌
def __init__(self, p1, p2, p3, p4, p5, p6, p7, p8, p9, p10):
    pass

# ✅ Usar dataclass o objeto de configuración
from dataclasses import dataclass

@dataclass
class ConfiguracionApp:
    host: str
    puerto: int
    debug: bool
    timeout: int = 30

def __init__(self, config: ConfiguracionApp):
    self.config = config
```

### **4. Números Mágicos**
```python
# ❌
if response.status_code == 200:
    pass
if edad >= 18:
    pass

# ✅
HTTP_OK = 200
EDAD_MINIMA_ADULTO = 18

if response.status_code == HTTP_OK:
    pass
if edad >= EDAD_MINIMA_ADULTO:
    pass
```

### **5. Comentarios Innecesarios**
```python
# ❌
# Suma dos números
def sumar(a, b):
    return a + b

# ✅ El nombre ya es claro
def sumar(a, b):
    return a + b

# ✅ Comenta solo lo complejo
def calcular_hash(password):
    # Usamos SHA-256 por requisitos de seguridad GDPR
    return hashlib.sha256(password.encode()).hexdigest()
```

## ✅ Checklist Código de Calidad

- [ ] Cada clase tiene una responsabilidad clara
- [ ] Nombres descriptivos (variables, funciones, clases)
- [ ] Métodos cortos (< 20 líneas idealmente)
- [ ] Pocos parámetros (< 5)
- [ ] Sin números mágicos (usar constantes)
- [ ] Validaciones en constructores
- [ ] Uso correcto de decoradores
- [ ] Código DRY (Don't Repeat Yourself)
- [ ] Fácil de testear
- [ ] Fácil de extender
- [ ] Documentación con docstrings
- [ ] Type hints cuando sea apropiado

## 💡 Consejos Finales

### **1. KISS (Keep It Simple, Stupid)**
```python
# Simplicidad > Complejidad
# Si algo se puede hacer simple, hazlo simple
```

### **2. YAGNI (You Aren't Gonna Need It)**
```python
# No programes para el futuro
# Implementa solo lo que necesitas AHORA
```

### **3. DRY (Don't Repeat Yourself)**
```python
# Si copias código 2 veces, extráelo a función/clase
```

### **4. Code Reviews**
```python
# Revisa tu código al día siguiente
# Te sorprenderás de lo que encuentres
```

### **5. Tests**
```python
# Código testeable = Código bien diseñado
# Si es difícil testear, probablemente está mal diseñado
```

## 🔧 Ejemplo: Refactoring Real

### **Antes:**
```python
def procesar_usuario(datos):
    # Validar email
    if "@" not in datos["email"]:
        return False
    
    # Validar edad
    if datos["edad"] < 18:
        return False
    
    # Guardar en BD
    conn = mysql.connector.connect(host="localhost", user="root")
    cursor = conn.cursor()
    cursor.execute(f"INSERT INTO usuarios VALUES ('{datos['nombre']}', '{datos['email']}')")
    conn.commit()
    
    # Enviar email
```

### **Después (SOLID + Buenas Prácticas):**
```python
from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass
class Usuario:
    nombre: str
    email: str
    edad: int

class Validador:
    @staticmethod
    def validar_email(email: str) -> bool:
        return "@" in email and "." in email
    
    @staticmethod
    def validar_edad_adulto(edad: int) -> bool:
        EDAD_MINIMA = 18
        return edad >= EDAD_MINIMA

class RepositorioUsuarios(ABC):
    @abstractmethod
    def guardar(self, usuario: Usuario) -> bool:
        pass

class RepositorioMySQL(RepositorioUsuarios):
    def __init__(self, config):
        self.config = config
    
    def guardar(self, usuario: Usuario) -> bool:
        # Lógica MySQL con parámetros seguros
        pass

class ServicioEmail(ABC):
    @abstractmethod
    def enviar_bienvenida(self, email: str) -> bool:
        pass

class ServicioSMTP(ServicioEmail):
    def __init__(self, config):
        self.config = config
    
    def enviar_bienvenida(self, email: str) -> bool:
        # Lógica email
        pass

class ProcesadorUsuarios:
    def __init__(self, 
                 repositorio: RepositorioUsuarios,
                 servicio_email: ServicioEmail):
        self.repositorio = repositorio
        self.servicio_email = servicio_email
    
    def procesar(self, usuario: Usuario) -> bool:
        # Validar
        if not Validador.validar_email(usuario.email):
            return False
        if not Validador.validar_edad_adulto(usuario.edad):
            return False
        
        # Guardar
        if not self.repositorio.guardar(usuario):
            return False
        
        # Notificar
        return self.servicio_email.enviar_bienvenida(usuario.email)

# Uso
repo = RepositorioMySQL(config_bd)
email_service = ServicioSMTP(config_email)
procesador = ProcesadorUsuarios(repo, email_service)

usuario = Usuario("Alvaro", "alvaro@example.com", 25)
procesador.procesar(usuario)
```
