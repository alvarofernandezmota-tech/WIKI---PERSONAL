# 1.1 Programación Estructurada vs POO

## 📄 Programación Estructurada

### Definición
Paradigma que organiza el código en secuencias lógicas, usando:
- Variables
- Funciones
- Estructuras de control (if, for, while)

### Ejemplo en Python:
```python
# Programación Estructurada
citas = []

def agregar_cita(nombre, telefono, fecha, hora):
    cita = {
        'nombre': nombre,
        'telefono': telefono,
        'fecha': fecha,
        'hora': hora
    }
    citas.append(cita)

def ver_citas():
    for cita in citas:
        print(cita)

# Uso
agregar_cita("Juan", "612345678", "15/02/2026", "10:00")
ver_citas()
```

### Características:
- ✅ Simple y directa
- ✅ Fácil de entender para programas pequeños
- ❌ Difícil de mantener en proyectos grandes
- ❌ Datos y funciones separados

---

## 📦 Programación Orientada a Objetos (POO)

### Definición
Paradigma que organiza el código en **objetos** que tienen:
- **Atributos** (datos)
- **Métodos** (funciones)

### Ejemplo en Python:
```python
# Programación Orientada a Objetos
class Cita:
    def __init__(self, nombre, telefono, fecha, hora):
        self.nombre = nombre
        self.telefono = telefono
        self.fecha = fecha
        self.hora = hora
    
    def mostrar(self):
        print(f"Cita: {self.nombre} - {self.fecha} {self.hora}")

class GestorCitas:
    def __init__(self):
        self.citas = []
    
    def agregar_cita(self, cita):
        self.citas.append(cita)
    
    def ver_citas(self):
        for cita in self.citas:
            cita.mostrar()

# Uso
gestor = GestorCitas()
cita1 = Cita("Juan", "612345678", "15/02/2026", "10:00")
gestor.agregar_cita(cita1)
gestor.ver_citas()
```

### Características:
- ✅ Organizado y modular
- ✅ Fácil de mantener y escalar
- ✅ Reutilizable
- ✅ Datos y comportamiento juntos
- ❌ Más complejo al principio

---

## 🔄 Comparación

| Aspecto | Estructurada | POO |
|---------|--------------|-----|
| **Organización** | Funciones + datos separados | Objetos (datos + métodos) |
| **Complejidad** | Simple | Más compleja |
| **Mantenimiento** | Difícil en proyectos grandes | Fácil |
| **Reutilización** | Limitada | Alta (herencia) |
| **Ejemplo** | THDORA Fase 1-3 | THDORA Fase 4+ |

---

## 🎯 ¿Cuándo usar cada una?

### Usa Programación Estructurada cuando:
- ✅ Proyecto pequeño o script simple
- ✅ Prototipo rápido
- ✅ Estás aprendiendo a programar

### Usa POO cuando:
- ✅ Proyecto mediano o grande
- ✅ Código que crecerá con el tiempo
- ✅ Múltiples desarrolladores
- ✅ Necesitas reutilizar código

---

## 💡 Analogía

### Programación Estructurada:
**Como una lista de tareas:**
1. Comprar ingredientes
2. Cocinar
3. Servir

### POO:
**Como una cocina con objetos:**
- **Chef** (clase): tiene nombre, especialidad, puede cocinar()
- **Receta** (clase): tiene ingredientes, pasos, puede preparar()
- **Plato** (clase): tiene nombre, precio, puede servir()

---

## 📝 Resumen

- **Estructurada**: Funciones que operan sobre datos
- **POO**: Objetos que encapsulan datos y comportamiento
- **Python soporta ambas**: Puedes mezclarlas
- **THDORA**: Empezamos estructurada, evolucionamos a POO
