# 1.2 Los 4 Principios de la POO

## 🎯 Principios Fundamentales

La POO se basa en 4 pilares fundamentales:

1. **Encapsulación** 🔒
2. **Abstracción** 🎨
3. **Herencia** 🧱
4. **Polimorfismo** 🦉

---

## 1️⃣ Encapsulación 🔒

### Definición
**Ocultar los detalles internos y exponer solo lo necesario.**

### Analogía
Un **coche**: 
- Usas el volante, pedales, cambio (interfaz pública)
- NO ves cómo funciona el motor internamente (detalles ocultos)

### Ejemplo en Python:
```python
class CuentaBancaria:
    def __init__(self, titular, saldo_inicial):
        self.titular = titular  # Público
        self.__saldo = saldo_inicial  # Privado (doble guion bajo)
    
    def depositar(self, cantidad):
        """Método público"""
        if cantidad > 0:
            self.__saldo += cantidad
            return True
        return False
    
    def ver_saldo(self):
        """Acceso controlado al saldo"""
        return self.__saldo

# Uso
cuenta = CuentaBancaria("Juan", 1000)
cuenta.depositar(500)
print(cuenta.ver_saldo())  # 1500
# print(cuenta.__saldo)  # ❌ ERROR: Atributo privado
```

### Beneficios:
- ✅ Protección de datos
- ✅ Control de acceso
- ✅ Fácil de modificar internamente sin afectar código externo

---

## 2️⃣ Abstracción 🎨

### Definición
**Mostrar solo las características esenciales, ocultando la complejidad.**

### Analogía
Un **control remoto**:
- Botones simples: encender, cambiar canal, volumen
- Complejidad interna oculta: electrónica, señales infrarrojas

### Ejemplo en Python:
```python
from abc import ABC, abstractmethod

class Vehiculo(ABC):
    """Clase abstracta"""
    
    @abstractmethod
    def arrancar(self):
        """Método abstracto - debe implementarse"""
        pass
    
    @abstractmethod
    def detener(self):
        pass

class Coche(Vehiculo):
    def arrancar(self):
        return "Coche arrancado con llave"
    
    def detener(self):
        return "Coche detenido con freno"

class Moto(Vehiculo):
    def arrancar(self):
        return "Moto arrancada con botón"
    
    def detener(self):
        return "Moto detenida con freno"

# Uso
coche = Coche()
print(coche.arrancar())  # "Coche arrancado con llave"
```

### Beneficios:
- ✅ Interfaz simple
- ✅ Reduce complejidad
- ✅ Enfoque en lo importante

---

## 3️⃣ Herencia 🧱

### Definición
**Una clase puede heredar atributos y métodos de otra clase.**

### Analogía
**Genética**:
- Hijo hereda características de los padres
- Puede tener características propias

### Ejemplo en Python:
```python
class Persona:
    """Clase padre"""
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad
    
    def saludar(self):
        return f"Hola, soy {self.nombre}"

class Estudiante(Persona):
    """Clase hija - hereda de Persona"""
    def __init__(self, nombre, edad, carrera):
        super().__init__(nombre, edad)  # Heredar constructor
        self.carrera = carrera  # Atributo propio
    
    def estudiar(self):
        return f"{self.nombre} está estudiando {self.carrera}"

class Profesor(Persona):
    """Otra clase hija"""
    def __init__(self, nombre, edad, materia):
        super().__init__(nombre, edad)
        self.materia = materia
    
    def enseñar(self):
        return f"{self.nombre} enseña {self.materia}"

# Uso
estudiante = Estudiante("Ana", 20, "Ingeniería")
profesor = Profesor("Juan", 45, "Matemáticas")

print(estudiante.saludar())  # Heredado de Persona
print(estudiante.estudiar())  # Propio de Estudiante
print(profesor.enseñar())  # Propio de Profesor
```

### Beneficios:
- ✅ Reutilización de código
- ✅ Jerarquía lógica
- ✅ Fácil mantenimiento

---

## 4️⃣ Polimorfismo 🦉

### Definición
**La misma interfaz puede tener múltiples formas/implementaciones.**

### Analogía
**Un botón de play**:
- En música: reproduce canción
- En video: reproduce película
- En juego: inicia partida

### Ejemplo en Python:
```python
class Animal:
    def hacer_sonido(self):
        pass

class Perro(Animal):
    def hacer_sonido(self):
        return "🐶 Guau guau"

class Gato(Animal):
    def hacer_sonido(self):
        return "🐱 Miau miau"

class Vaca(Animal):
    def hacer_sonido(self):
        return "🐄 Muuu"

# Polimorfismo en acción
def presentar_animal(animal):
    """Misma función, diferentes comportamientos"""
    print(animal.hacer_sonido())

# Uso
animales = [Perro(), Gato(), Vaca()]

for animal in animales:
    presentar_animal(animal)
# Output:
# 🐶 Guau guau
# 🐱 Miau miau
# 🐄 Muuu
```

### Beneficios:
- ✅ Flexibilidad
- ✅ Extensibilidad
- ✅ Código más limpio

## 📊 Resumen Visual

```
🔒 ENCAPSULACIÓN
   ├─ Ocultar detalles internos
   └─ Exponer solo lo necesario

🎨 ABSTRACCIÓN
   ├─ Interfaz simple
   └─ Complejidad oculta

🧱 HERENCIA
   ├─ Clase padre → Clase hija
   └─ Reutilizar código

🦉 POLIMORFISMO
   ├─ Misma interfaz
   └─ Múltiples implementaciones
```

## 🎯 Aplicación en THDORA

### Con POO:
```python
class Cita:  # Encapsulación
    def __init__(self, nombre, telefono, fecha):
        self.__nombre = nombre  # Privado
        self.fecha = fecha  # Público

class CitaDental(Cita):  # Herencia
    def __init__(self, nombre, telefono, fecha, tratamiento):
        super().__init__(nombre, telefono, fecha)
        self.tratamiento = tratamiento

class CitaLimpieza(Cita):  # Herencia
    def __init__(self, nombre, telefono, fecha):
        super().__init__(nombre, telefono, fecha)
        self.tipo = "Limpieza"

# Polimorfismo
def procesar_cita(cita):
    cita.confirmar()  # Mismo método, diferente comportamiento
```
