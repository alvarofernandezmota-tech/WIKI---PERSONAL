## ⚠️ Problema UFW detectado

**Las reglas `192.168.1.0/24` son incorrectas** — la red real de madre es `10.48.234.0/24`, no `192.168.1.0/24`.
Esto significa que los servicios estaban efectivamente accesibles desde `0.0.0.0` (cualquier IP).

### Reglas incorrectas a eliminar (números 2-15):
```bash
# Borrar de mayor a menor para que no se reordenen mal:
for n in 15 14 13 12 11 10 9 8 7 6 5 4 3 2; do
  sudo ufw delete $n
done
```

### Reglas correctas añadidas:
- `10.48.234.0/24` → red upstream Redmi (USB tethering)
- `192.168.72.0/24` → red MadreAP WiFi
- `100.64.0.0/10` → red Tailscale
- `24800` DENY → input-leaps cerrado
- `5355` DENY → mDNS cerrado
- `2222` solo LAN+Tailscale → docker SSH restringido

## 🛠 Script hardening completo

Guardado en: `scripts/hardening-ufw.sh`

```bash
#!/bin/bash
# hardening-ufw.sh — Corregir UFW madre
# Ejecutar desde madre con sudo

# 1. Borrar reglas incorrectas 192.168.1.0/24 (ajustar números según ufw status numbered):
for n in 15 14 13 12 11 10 9 8 7 6 5 4 3 2; do
  sudo ufw delete $n
done

# 2. Añadir reglas correctas para todos los servicios:
REDES="10.48.234.0/24 192.168.72.0/24 100.64.0.0/10"
PUERTOS="3000 3001 3002 3003 5001 5678 6333 6334 6901 8000 8443 9000 9090 11434 11435 19999 2222"

for red in $REDES; do
  for puerto in $PUERTOS; do
    sudo ufw allow from $red to any port $puerto
  done
done

# 3. Cerrar puertos peligrosos:
sudo ufw deny 24800  # input-leaps
sudo ufw deny 5355   # mDNS

# 4. Verificar:
sudo ufw status verbose
```
