## 🖥️ Servidor

**Sesión completa de diseño e infraestructura — de cero a Fase 1 en ejecución.**

### Decisión de arquitectura fijada

- **Madre = cerebro** — todo lo que requiere GPU o presencia humana
- **Acer = soporte dentro Y fuera** — servicios 24/7 + puerta exterior vía Tailscale
- **MacBook = cliente puro** — consume, no aloja nada

Cuando estés fuera de casa: entras al Acer vía Tailscale, no al MacBook.

### Servicios por máquina

| Servicio | Máquina | Motivo |
|---|---|---|
| Ollama + Open WebUI | Madre | GTX 1060 solo aquí |
| Input Leap server | Madre | Emite desde el cerebro |
| THDORA + PostgreSQL + Pi-hole | Acer | 24/7 sin intervención |
| Tailscale | Madre + Acer | IPs fijas en toda la red |
| Acceso remoto exterior | Acer vía Tailscale | Puerta única |

### Incidencia resuelta: NeedsLogin

El proceso `tailscale up` se canceló antes de completar el login web.
`tailscale ip -4` devolvió `state: NeedsLogin`.

**Fix documentado:**
```bash
sudo tailscale up --force-reauth
```
Abrir el enlace generado → autorizar en panel web → verificar con `tailscale ip -4`.

## 📚 Formación

Aprendido hoy:

- **Tailscale vs Headscale** — cliente OS vs servidor de coordinación self-hosted
- **Zero Trust en LAN** — UFW lista blanca de IPs Tailscale
- **systemd services** — `enable --now` vs arrancar manualmente
- **NeedsLogin fix** — `--force-reauth` regenera el enlace de autenticación

Concepto clave:
