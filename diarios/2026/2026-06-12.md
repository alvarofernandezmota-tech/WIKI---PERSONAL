## 🖥️ Servidor — FASE 1 COMPLETADA

### IPs Tailscale (permanentes)

| Máquina | IP Tailscale | Entorno | Rol |
|---|---|---|---|
| **Madre** | `100.91.112.32` | Hyprland / Wayland | Input Leap SERVER |
| **Acer** | `100.86.119.102` | Wayland | Input Leap CLIENT |

### Lo ejecutado en Madre

```bash
sudo pacman -Syu tailscale --noconfirm
sudo systemctl enable --now tailscaled.service
sudo tailscale up --force-reauth   # fix NeedsLogin
tailscale ip -4                    # → 100.91.112.32

mkdir -p ~/.config/input-leap
# input-leap.conf: madre(izq) → acer(dcha)

mkdir -p ~/.config/systemd/user
# input-leaps.service: QT_QPA_PLATFORM=wayland, 0.0.0.0:24800, -n madre
systemctl --user enable --now input-leaps.service
```

### Lo ejecutado en Acer (vía SSH desde Madre)

```bash
ssh varo@100.86.119.102

sudo pacman -Syu tailscale --noconfirm
sudo systemctl enable --now tailscaled.service
sudo tailscale up
tailscale ip -4   # → 100.86.119.102

# input-leapc.service: -n acer 100.91.112.32:24800
systemctl --user enable --now input-leapc.service

# UFW Zero Trust
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow from 100.91.112.32 to any port 22 proto tcp
sudo ufw allow from 100.91.112.32 to any port 24800 proto tcp
sudo ufw enable
```

### Incidencia resuelta

- `tailscale up` cancelado con `^C` antes de auth → estado `NeedsLogin`
- Fix: `sudo tailscale up --force-reauth` → nuevo enlace → autorizado en panel Tailscale

### Decisión de arquitectura fijada

- **Madre = cerebro** — workstation + Hyprland + GPU + Input Leap server
- **Acer = soporte 24/7** — servicios continuos + puerta exterior vía Tailscale
- **MacBook = cliente puro** — consume servicios, no expone nada
- Docker preexistente en Acer — reglas UFW propias preservadas (`172.x`, `192.x`, `53317`)

## 📚 Formación — aprendido hoy

- **Tailscale en Arch Linux** — `pacman`, `systemctl enable --now`, flujo de auth
- **`--force-reauth`** — regenera el enlace de auth sin reinstalar
- **Input Leap en Wayland** — `QT_QPA_PLATFORM=wayland` obligatorio en el unit file
- **systemd usuario vs sistema** — `systemctl --user` para servicios de sesión gráfica
- **UFW Zero Trust** — `deny incoming` por defecto + whitelist por IP Tailscale
- **SSH por IP Tailscale** — más estable que IP LAN DHCP (puede cambiar con DHCP)
- **IaC personal** — toda config del servidor versionada en Git

**Concepto clave del día:**
> *"UFW con `deny incoming` por defecto + whitelist por IP Tailscale = Zero Trust real. Nadie entra si no está en la lista. Eso es lo que separa un servidor serio de uno expuesto."*
