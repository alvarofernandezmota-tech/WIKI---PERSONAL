## 🖥️ Servidor

**Sesión completa de diseño e infraestructura — de cero a arquitectura documentada.**

### Decisión de arquitectura fijada

Despues de analizar los recursos y los roles de cada máquina:

- **Madre = cerebro** — todo lo que requiere GPU o presencia humana
- **Acer = soporte dentro Y fuera** — servicios 24/7 + puerta de entrada exterior vía Tailscale
- **MacBook = cliente puro** — consume, no aloja nada

Cuando estés fuera de casa: entras al Acer vía Tailscale, no al MacBook.
El Acer es el servidor real — interno y externo.

### Servicios decididos por máquina

| Servicio | Máquina | Motivo |
|---|---|---|
| Ollama + Open WebUI | Madre | GTX 1060 — solo está aquí |
| Input Leap server | Madre | Emite desde el cerebro |
| THDORA + PostgreSQL + Pi-hole | Acer | 24/7 sin intervención |
| fail2ban + logs | Acer | Seguridad siempre activa |
| Tailscale | Madre + Acer | IPs fijas en toda la red |
| Acceso remoto exterior | Acer vía Tailscale | Puerta de entrada única |

### Roadmap del servidor

```
FASE 1 — Conectividad (PRÓXIMO PASO)
  ├── Tailscale en Madre + Acer → IPs 100.x.x.x estables
  ├── SSH Madre → Acer
  └── Input Leap server en Madre, client en Acer+Mac, UFW Zero Trust

FASE 2 — Seguridad
  ├── TLS en Input Leap
  ├── fail2ban en Acer
  └── Headscale self-hosted

FASE 3 — Servicios
  ├── Ollama + Open WebUI en Madre
  ├── PostgreSQL + THDORA + Pi-hole en Acer
```

## 📚 Formación

Aprendido y fijado hoy:

- **Tailscale vs Headscale** — cliente OS vs servidor de coordinación (Headscale = Fase 2)
- **Zero Trust en LAN** — UFW lista blanca de IPs Tailscale, no puertos abiertos al mundo
- **systemd para servicios** — diferencia entre "programa en terminal" y "servicio de producción"
- **Arquitectura cerebro+soporte** — separar workstation de servicios 24/7 quita carga real a Madre
- **IaC personal** — toda config del servidor bajo Git = Infrastructure as Code
- **Repo como base de datos** — carpeta=tabla, archivo=registro, sección=campo

Concepto clave del día:
> *"Madre no debe saber que el Acer existe en términos de carga. Solo le envía el teclado y le manda comandos por SSH. El resto lo hace el Acer solo."*
